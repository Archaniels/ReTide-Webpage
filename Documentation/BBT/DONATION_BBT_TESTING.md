# Donation Feature - Black Box Testing Suite

## Overview
This document outlines the Black Box Testing suite for the **Donation** feature. The test cases have been meticulously designed utilizing **Equivalence Class Partitioning (EQP)** and **Boundary Value Analysis (BVA)** to ensure comprehensive coverage of valid, invalid, and edge-case scenarios.

### Assumed Validation Constraints
To properly apply EQP and BVA methodologies, the following standard constraints are assumed for the donation form fields:
*   **Donation Amount**: Numeric decimal field. Minimum allowed = `$1.00`, Maximum allowed = `$10,000.00`.
*   **Donor Name**: String field. Required. Minimum length = `2`, Maximum length = `50`. Must contain only alphabetic characters, spaces, and hyphens.
*   **Email Address**: String field. Required. Must conform to standard email format (e.g., `user@domain.tld`).
